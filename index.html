What's a token, and how does pricing work?

A token is a chunk of text — roughly ¾ of a word in English. Models read and write text as tokens, not characters, so providers bill per token rather than per character.

Pricing is quoted per 1M tokens and usually split into an input rate (what you send) and a higher output rate (what the model generates). This tool multiplies your counted tokens by the selected model's per-million rate and divides by 1,000,000.

Tokenization for GPT-4o, GPT-4o mini, GPT-4 Turbo, and GPT-3.5 Turbo runs via js-tiktoken, the same BPE encoding those models use in production — so those counts are exact. GPT-5.6 uses OpenAI's newer o200k_harmony tokenizer, which isn't yet available in the browser library this tool uses, so its counts are estimated. Anthropic and Google also haven't published browser-usable tokenizers for Claude or Gemini. All estimated counts use a ~4-characters-per-token approximation and are labeled accordingly.
